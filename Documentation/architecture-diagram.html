  • deleteAlbum() → DELETE

6. Anti-Corruption Layer (ACL)

Boundaries that prevent legacy data model artifacts from leaking into the modernized system

_class Field

"_class": "org.cloudfoundry...Album"
↓ strip on import ↓
(field removed, never exposed)

PUT/POST Semantics

PUT = create, POST = update (unconventional)
↓ preserve at boundary ↓
PUT = create, POST = update (contract compat)

Null for Missing

GET /albums/{id} → .orElse(null)
↓ match at API boundary ↓
GET /albums/:id → res.json(null)

CF-Specific Data

CfEnv.findAllServices() → profiles[]
↓ adapt ↓
{ profiles: ["node"], services: [] }

Field Casing

releaseYear, trackCount, albumId
↓ keep exact casing ↓
releaseYear, trackCount, albumId

albumId vs id

Both id (PK) and albumId exist on entity
↓ preserve, decouple ↓
id = PK (UUID), albumId = legacy compat field
7. Request Lifecycle — Add Album
Frontend User clicks "Add an album"
AlbumsPage opens <AlbumForm action="add" />
Frontend User fills form & clicks OK
AlbumForm validates: title (req), artist (req), releaseYear (/^[1-2]\d{3}$/), genre (req)
Frontend useAlbums hook calls API
api.addAlbum(album) → PUT /albums { title, artist, releaseYear, genre }
Backend Express receives PUT /albums
routes/albums.js → validation middleware checks body
Backend Model saves to database
models/album.js.save() → generates UUID → INSERT INTO albums
Database SQLite inserts row
INSERT INTO albums (id, title, artist, release_year, genre, ...) VALUES (...)
Backend Returns saved album
res.json({ id: "uuid-...", title, artist, releaseYear, genre, trackCount, albumId })
Frontend Hook updates state
useAlbums: re-fetches album list • useStatus: success("Album saved")
Frontend UI re-renders
AlbumGrid shows new card • StatusMessage shows green "Album saved" alert
